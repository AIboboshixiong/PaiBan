# 架构设计：赛博公众号排版转换器 (CyberGongZH)

## 1. 技术栈锁定 (准则 3)
*   **前端**: React 18.2.0, Vite 5.0.0
*   **样式**: CSS Modules + Vanilla CSS (赛博蓝色系: #38bdf8, #00d2ff, #020617)
*   **状态管理**: React Hooks
*   **包管理**: pnpm 10.14.0
*   **关键依赖**: 
    *   `lucide-react`: 0.243.0
    *   `framer-motion`: 10.16.4
    *   `copy-to-clipboard`: 3.3.3

## 2. 核心功能设计
*   **输入端**: 多行文本输入框，支持 `#` (一级标题) 和 `##` (二级标题) 识别。
*   **解析引擎**: 基于正则表达式的轻量化 Markdown 解析，将其转换为带内联样式的 HTML。
*   **样式矩阵**:
    *   **样式 A (霓虹边缘)**: 带有左侧发光条和斜面背景。
    *   **样式 B (量子节点)**: 带有点状装饰和底部渐变线。
    *   **样式 C (矩阵网格)**: 带有网格背景和方框标记。
*   **输出端**: 一键将预览区的 **计算后带内联样式 (Inline-style)** 的 HTML 复制到剪贴板，确保微信公众号编辑器能够完全识别并保留样式。

## 3. 目录结构 (准则 2)
```text
/
├── .temp-dev/             # 临时文件与原型 (准则 16)
│   └── prototype/         # 静态预览 HTML
├── src/
│   ├── components/        # 核心组件 (Editor, Preview, ThemeSelector)
│   ├── styles/            # 赛博视觉变量定义
│   └── utils/             # HTML 转换助手 (样式注入逻辑)
├── scripts/               # 自动安装与验证脚本 (准则 15)
└── 架构设计.md            # 本文档
```

## 4. 命名规范 (准则 4 & 20)
*   **词根**: CyberHeader, CyberSubHeader, CyberBody, ThemeColor.
*   **格式**: TS/JS 使用 camelCase, 组件使用 PascalCase, CSS 使用 kebab-case。
